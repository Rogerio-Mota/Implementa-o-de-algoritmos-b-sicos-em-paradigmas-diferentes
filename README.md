O paradigma Orientado a Objetos (POO) é o mais fácil de entender para esse problema porque organiza o código de forma parecida com a vida real. Em vez de trabalhar só com listas e comandos, criamos objetos que representam coisas do sistema, como um Funcionário com nome e salário, e uma Folha de Pagamento que calcula os bônus.
Essa organização deixa o código mais claro, organizado e fácil de entender. Enquanto o modelo Imperativo foca em como fazer as tarefas e o Funcional em transformar dados, a POO foca em quem faz cada coisa dentro do sistema, o que acaba sendo mais natural para a forma como pensamos.
